# 🧠 JavaScript Project Understanding Checklist

Use this while building ANY project (calculator, todo app, weather app, etc.)

---

# 🟢 1. Can I explain the app?

- [ ] Can I explain what the app does in 1–2 sentences?
- [ ] Can I explain it to a beginner without using code?

If NO → I don’t fully understand the goal yet

---

# 🟡 2. Do I understand the data?

- [ ] What data does my app store?
- [ ] Where is it stored? (array, object, variables, etc.)
- [ ] What does one “item” look like?

Example check:

- “I know what a task looks like”
- “I know what a weather response contains”

---

# 🟡 3. Do I understand user input?

- [ ] What does the user do? (click, type, select)
- [ ] How do I get that input in JS?
- [ ] What happens immediately after input?

---

# 🟡 4. Do I understand the flow?

- [ ] Can I describe step-by-step what happens?
      (Example: click → get input → update data → update UI)

If I cannot explain the flow → I’m guessing, not understanding

---

# 🟡 5. Do I understand the UI update?

- [ ] Where does the HTML come from?
- [ ] How does the screen change when data changes?
- [ ] Do I rebuild the UI or manually change parts?

Key idea:

- Data changes → UI updates

---

# 🟡 6. Do I understand each feature?

For every feature (add, delete, mark done):

- [ ] Can I explain how it works in words?
- [ ] Can I trace it step-by-step?

If NO → that feature needs review

---

# 🔴 7. Can I rebuild it?

- [ ] Could I rebuild this project without a tutorial?
- [ ] Could I do it again from scratch (even slowly)?

If NO → I should review, not move on yet

---

# 🟢 8. Can I explain it out loud?

- [ ] Can I explain the whole project like I’m teaching someone?
- [ ] Do I get stuck explaining anything?

If YES → that part is what I need to improve

---

# 🚀 FINAL RULE

> If I can explain it simply → I understand it  
> If I cannot → I still need to learn that part

---

# ⚠️ IMPORTANT REMINDER

You do NOT need:

- perfect code
- memorized syntax
- advanced features

You DO need:

- understanding of flow
- understanding of data
- ability to explain
